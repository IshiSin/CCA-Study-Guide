## Study Guide Sections

This guide is built around five complementary learning modes. Use all of them — they reinforce each other.

| Section | What it's for | Time/session |
|---------|--------------|-------------|
| **Study** | In-depth topic coverage | 20–30 min |
| **Quiz** | Test recall, find gaps | 15–20 min |
| **Scenarios** | Apply knowledge to real systems | 20–30 min |
| **Flashcards** | Rapid review, spaced repetition | 10–15 min |
| **Cheatsheets** | Quick reference before exam | 10 min |
| **Labs** | Hands-on Python practice | 30–60 min |

## Effective Study Strategies

### 1. Read → Quiz → Re-read

Don't quiz yourself before you've read the topic at least once. The correct pattern:

1. Read the study topic fully
2. Take the domain quiz immediately after
3. For every question you got wrong, re-read that section

### 2. Anti-patterns first

When you start a new domain, read the **Anti-Patterns** topic first. Knowing what's wrong makes the right answers obvious.

<ExamTip>
Many exam questions are eliminated by asking: "Is this achieving the goal through configuration/code, or through hoping the model follows instructions?" If it's the latter, it's usually the wrong answer.
</ExamTip>

### 3. Use flashcards daily

Spend 10–15 minutes on flashcards each day, not just before the exam. Spaced repetition dramatically improves retention for concept-heavy domains like MCP and context management.

### 4. Do the labs

The labs in `/local/` are Python exercises you run locally. They're not required, but candidates who complete them consistently report that scenario-based questions become much easier — you've *seen* the patterns in actual code.

<Callout variant="tip" title="Lab setup">
See `local/README.md` for how to install the Python dependencies and run labs. You'll need an Anthropic API key.
</Callout>

### 5. Time yourself on mock exams

The 120-minute limit works out to 2 minutes per question. Practice under time pressure at least once before the real exam. The mock exam at `/quiz/exam` has a timer.

<KeyTakeaway>
- Read → Quiz → Re-read is more effective than read-only study
- Anti-patterns are the highest-leverage thing to learn in each domain
- Do flashcards daily (10–15 min), not just before the exam
- Complete at least one timed mock exam before your test date
- Labs are optional but make scenario questions much easier
</KeyTakeaway>
