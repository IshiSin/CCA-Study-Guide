The **context window** is the total amount of text — measured in tokens — that a model can see at once. Everything that influences a Claude response must fit: the system prompt, conversation history, tool results, file contents, and the response itself. When context fills up, you cannot simply add more — something must be removed or summarized.

200K tokens sounds large, but tool-heavy agentic sessions consume it quickly. A single large file read can consume tens of thousands of tokens.

<ExamTip>
The exam tests that you know the context window is **shared** between input and output — it is not "200K input + separate output." The `max_tokens` parameter controls how much of the window Claude can use for its response. If you set `max_tokens=4096` with a 195K-token prompt, only 4K tokens remain for the response.
</ExamTip>

## Token Counting

Tokens are not characters or words — they are subword units from Claude's tokenizer. As a rough rule:

- **1 token ≈ 4 characters of English prose**
- **1 token ≈ ¾ of a word**
- **Code tokenizes differently** — punctuation, identifiers, and whitespace each cost tokens

```python
import anthropic

client = anthropic.Anthropic()

# Count tokens before sending — avoid surprise context overflows
response = client.messages.count_tokens(
    model="claude-opus-4-6",
    system="You are a helpful assistant.",
    messages=[
        {"role": "user", "content": "Explain the context window."}
    ]
)

print(f"Input tokens: {response.input_tokens}")
# Use this to gate requests: if tokens > threshold, summarize first
```

## How Claude Code Manages Context

Claude Code tracks context usage and applies strategies automatically as sessions grow:

### Auto-Compaction (`/compact`)

When context nears the limit, Claude Code can summarize the conversation to free space:

```
User: /compact

Claude: I'll summarize our conversation so far to free up context space.

[Summary of what we've worked on, decisions made, current state of files]

Context has been compacted. Continuing from the current state of the codebase.
```

The `/compact` command can also be invoked manually at any time. After compaction:
- Prior conversation turns are replaced by a structured summary
- File edits made during the session are preserved (they're on disk)
- The model retains the work context without the full verbatim history

### What Claude Code Prioritizes in Context

When managing context, Claude Code keeps:
1. **System prompt and CLAUDE.md** — always present
2. **Recent turns** — the last N exchanges are kept verbatim
3. **Summaries of older turns** — earlier history is compressed
4. **Current file states** — re-read as needed rather than kept in context

## Context Budget Planning

For long agentic tasks, budget context explicitly:

```python
import anthropic

client = anthropic.Anthropic()

CONTEXT_LIMIT = 200_000
RESPONSE_BUDGET = 8_000       # Reserve for Claude's response
SAFETY_MARGIN = 10_000        # Don't fill to the brim
USABLE_BUDGET = CONTEXT_LIMIT - RESPONSE_BUDGET - SAFETY_MARGIN  # 182,000

def build_prompt_with_budget(files: list[str], task: str) -> list[dict]:
    """Include as many files as fit within the context budget."""
    messages = []
    tokens_used = 0

    for path in files:
        with open(path) as f:
            content = f.read()

        file_message = {"role": "user", "content": f"File: {path}\n\n{content}"}

        # Count tokens for this file
        token_count = client.messages.count_tokens(
            model="claude-opus-4-6",
            messages=[file_message],
        ).input_tokens

        if tokens_used + token_count > USABLE_BUDGET:
            # Would overflow — stop adding files
            messages.append({
                "role": "user",
                "content": f"[Note: {len(files) - len(messages)} files omitted due to context limits]"
            })
            break

        messages.append(file_message)
        tokens_used += token_count

    messages.append({"role": "user", "content": task})
    return messages
```

## Context Strategies for Large Codebases

When the codebase is too large to fit in context:

**1. Selective inclusion — only read what's relevant**
```python
# Instead of reading all files upfront, read as needed during the task
# Let Claude decide what to read based on the task
system = """When you need to understand a file, use the Read tool to fetch it.
Don't try to hold the entire codebase in memory — explore lazily."""
```

**2. Summarize before including**
```python
def summarize_file_for_context(path: str) -> str:
    """Get a compact summary instead of raw file content."""
    with open(path) as f:
        content = f.read()

    # Ask Claude to summarize the file first
    summary = client.messages.create(
        model="claude-haiku-4-5",  # Use cheap model for summarization
        max_tokens=500,
        messages=[{
            "role": "user",
            "content": f"Summarize this file in 3-5 sentences focusing on: what it exports, its dependencies, and any important patterns.\n\n{content}"
        }]
    )
    return f"[{path}]: {summary.content[0].text}"
```

**3. Use retrieval instead of injection**
```python
# Build a vector index of the codebase
# Retrieve only the N most relevant chunks for each task
# Avoids loading the entire codebase into context
relevant_chunks = vector_store.search(query=task, top_k=10)
context = "\n\n".join(chunk.content for chunk in relevant_chunks)
```

<KeyTakeaway>
- Context window = system prompt + history + tool results + response — all share one 200K token budget
- `max_tokens` reserves output space; the remainder is available for input
- Use `client.messages.count_tokens()` to measure before sending to avoid overflows
- `/compact` summarizes conversation history to free context space
- For large tasks: read files lazily, summarize before including, or use retrieval-based selection
- Plan a safety margin (10K+) so the model isn't reasoning at the edge of its limit
</KeyTakeaway>
